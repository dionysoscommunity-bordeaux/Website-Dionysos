## Technique Execution Results

### First Principles Thinking — Fondations de marque

- **Compréhension immédiate + envie d'agir** : le site doit faire comprendre qui est Dionysos et donner envie de rejoindre.
- **Positionnement correct** : communauté 100% amicale (pas de dating).
- **Ton juste** : festif, chaleureux, inclusif, anti-forceur.
- **Promesse clé** : "sortir, voir du monde, passer du bon temps IRL" avec un onboarding progressif.
- **Principe UX structurant** : porte ouverte permanente (CTA Discord visible), sans pression.

### Role Playing — Parcours et preuves

- **Persona néo-Bordelais·e** : besoin de "gens comme moi", simplicité, zéro engagement.
- **Parcours validé** : types d'activités -> exemples concrets -> lien Discord.
- **Conversion naturelle** : le déclic arrive quand la personne reconnaît ce qu'elle cherche.
- **Partenaires** : ils veulent voir qui vous êtes, vos partenaires actuels, et comment vous contacter.
- **Mobile-first priorisé** : CTA sticky Discord > accroche courte > photos compressées > vidéo.
- **SEO réaliste** : pas de recap de chaque sortie; préférence pour pages piliers maintenables.
- **Couverture géographique** : Bordeaux global (pas logique par quartier dans la structure éditoriale).
- **Réassurance** : preuves via avis membres + partenaires.
- **RGPD** : pas de formulaire, redirection vers comptes personnels Discord/Instagram.
- **Accueil timide** : "Observe d'abord, participe à ton rythme."

### SCAMPER — Architecture et arbitrages

- **Substitute validés (5/5)** :
  - A propos long -> bloc "Qui on est / Ce qu'on fait / Rejoindre"
  - Formulaire -> portes directes Discord/Insta
  - Agenda exhaustif -> sélection vivante représentative
  - Preuve par chiffres -> preuve par retours + partenaires
  - CTA agressif -> CTA permanent discret
- **Architecture retenue** : Option 2 adaptée, sans page quartiers.
